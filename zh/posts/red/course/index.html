SICP#

We are about to study the idea of computational process
Processes manipulate other abstruct things called data
The evolution of a process is directed by a pattern of rules called program
People create programs to direct processes
Why lisp?
The most significant of these features is the fact that Lisp descriptions of processes,called procedures,can themselves be represented and manipulated as Lisp data.It has the ability to blur the tradional distinction between “passive” data and “active” processes.Programming in Lisp is great fun.
Every powerful languages has three mechanisms for accomplishing this

  • primitive expressions,which represent the simplest entities the language is concerned with
  • means of combination,by which compound elements are built from simpler ones
  • means of abstruction, by which compound elements can be named and manipulated as units.

In programming,we deal with two konds of elements:procedures and data

Expressions that formed by delimiting a list of expressions within parentheses in oredr to denote procedure application,are called combinations e.g. (+ 2 7)
the leftmost element in the list is called the operator,and the other elements are called operands,the value of a combination is obtained by applying the procedure specified by the operator to the arguments that are the values of the operands.
A critical aspect of a programming language is the means it provides fo using namnes to refer to computational objects.We say that the name identifies a variable whose value is the object. In Lisp,use define e.g. (define size 2) It should be clear that the possiblity of associating values with symbols and later retrieving them means that the interpreter must maintain some sort of memory that keeps track of the name-object pairs.The memory is called the environment
We have identified in Lisp some of the elements that must appear in any powerful programming language:

  • Numbers and arithmetic operations are primitive data and procedures
  • Nesting of combinations provides a means of combing operations
  • Defintions that associate names with values provide a limited means of abstraction

For compound procedures,the application process is as follows:
To apply a compound procedure to arguments,evaluate the body of the procedure with each formal parameter replaced by the corresponding argument.The process is called the substitution model for procedure application.In practice,the “substitution” is accomplished by using a local environment for the formal parameters.
We need to have a way to make tests and to perform different operations depending on the result of a test.This construct is called a case analysis,and in Lisp,called *cond*(which stands for “conditional”) e.g.

(cond (⟨p1⟩ ⟨e1⟩)
(⟨p2⟩ ⟨e2⟩)
. . .
(⟨pn⟩ ⟨en⟩)) 

(define (abs x)
        (cond ((> x 0) x)
              ((= x 0) 0)
              ((< x 0) (- x))))

(⟨p⟩ ⟨e⟩) called clauses,⟨p⟩ is a predicate
In Scheme,there are two distinguished values that are denoted by the constants #t and #f.When the interpreter checks a predicate’s value,it interprets #f as false.
there are logical composition operations,which enable us to construct compound predicates.such as

  • (and <e1> …)
  • (or <e1> … )
  • (not <e>)
