修改文件名#

以只读方式打开文件#
  • C x C r 新的窗口打开
  • C x 4 r 当前窗口打开

跳转#

书签#
  • C x r m 设置标签
  • C x r b 跳转到书签
  • C x r l 列出书签
register#

类似于书签，但是重启 emacs 会丢失，特点是可以精细保存到光标所在位

  • C x r space 设置
  • C x r j 跳转

窗口#

  • C x 2 类似划分窗口，同时打开当前的文件，向下分身
  • C x 3 向右边分身
  • C x 4 f 打开新的窗口 (find-file-other-window) 关于打开的窗口是水平还是垂直划分的，参考这里1
  • C x 4 C o 在新的窗口打开文件，但光标还在当前文件
  • C x 4 b 同上，但光标位置切换到新开的文件上
  • C x 1 删除其他窗口

恢复窗口布局，类似于 undo 和 redo 在窗口上的应用
使用一个包 winner-mode 

(use-package winner-mode
  :straight nil
  :hook (after-init . winner-mode))

移动#

  • M g g 移动到指定行
  • M v 向上滚动
  • C v 向下滚动
  • C M v 其他buffer向上滚动
  • C M S v 其他buffer向下滚动

删除#
  • C backspace 删除光标前的一个单词(无视空行)
  • M d 删除光标后的一个单词(无视空行)
  • C k 删除光标后的一行
  • C S k 删除光标前的一行

重新加载此页面#

一般当外界文件变动的时候emacs不能很好地实时变动
需要用到 revert-buffer 这个函数
每次执行的时候还需要确认,sf上有人提供了修改版的 

;; Source: https://www.emacswiki.org/emacs/misc-cmds.el
(defun revert-buffer-no-confirm ()
    "Revert buffer without confirmation."
    (interactive)
    (revert-buffer :ignore-auto :noconfirm))
C-k with no kill#
(defun delete-line-no-kill ()
  (interactive)
  (delete-region
   (point)
   (save-excursion (move-end-of-line 1) (point)))
  (delete-char 1)
  )
(global-set-key (kbd "C-k") 'delete-line-no-kill)

Todo#

将 heading 转换成 TODO 和切换 TODO 的状态可以用 C c C t 和 S 方向键
快速插入一个 TODO M S RET
TODO状态关键字可以自定义

  • 全局设定(在配置文件里添加以下代码)用竖线分隔已完成状态和未完成状态 
    (setq org-todo-keywords
        '((sequence "TODO" "FEEDBACK" "VERIFY" "|" "DONE" "DELEGATED")))
  • 单独文件设定
    在文件头添加这样的属性,同样用 | 隔开 
    #+TODO: TOPLAY PLAYING | CLEAR NEWGANEPLUS

设定关键字的样式 

(setq org-tode-keyword-faces
      '(("TODO" . "#ff9977")
  ("DONE" . "#44bc44")))

tricks#

导出org时没有换行#

org中每空一行表示一个段落,要想在段落里换行有以下方法

  • 在文件头部加上#+OPTIONS: \n:t
  • 或者在配置文件加上 
(setq org-export-preserve-breaks t)
  • 在换行处加上\\
刚打开时默认显示的标题级数#

两种方法

  • 在配置文件加上钩子(直接copy过来的,我没有定义快捷键,默认开启) 
    (defun org-show-two-levels ()
  (interactive)
  (org-content 2))

(add-hook 'org-mode-hook
          (lambda ()
            (define-key org-mode-map "\C-cm" 'org-show-two-levels)))
  • 在 org 头上可以加入下列指定语句(不过要注意,配置文件的优先级要高于此)
    • #+STARTUP show2levels (貌似可以指定到五级)
    • #+STARTUP content (以目录形式显示,即所有标题)
    • 还有 overview 和 showeverything 但每试过有啥用

  1. Normally this action is performed by the function split-window-sensibly, and controlled by two variables, split-height-threshold and split-width-threshold. Per the documentation

    If ‘split-height-threshold’ specifies an integer, WINDOW is at least ‘split-height-threshold’ lines tall and can be split vertically, split WINDOW into two windows one above the other and return the lower window. Otherwise, if ‘split-width-threshold’ specifies an integer, WINDOW is at least ‘split-width-threshold’ columns wide and can be split horizontally, split WINDOW into two windows side by side and return the window on the right. If this can’t be done either and WINDOW is the only window on its frame, try to split WINDOW vertically disregarding any value specified by ‘split-height-threshold’. If that succeeds, return the lower window. Return nil otherwise.

    On my system the default value for split-height-threshold is 80 and split-width-threshold is 160. I think you could get the effect you want by setting split-width-threshold to something lower. Alternatively you could set split-height-threshold to nil, preventing it from ever splitting vertically, or set the variable split-window-preferred-function to a different function. Any of these would affect, not just find-file-in-other-window, but all the functions that split the screen.  ↩︎
