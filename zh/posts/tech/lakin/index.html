工具学习#

git#

删除远程仓库的内容#

先要pull下来 删除文件或文件夹

  • 预览要删除的文件 git rm -r -n –cached 文件/文件夹名称
  • 确认无误后删除文件 git rm -r -cached 文件/文件夹名称

清空远程仓库

  • git rm -rf *
  • git commit -m ‘Empty the repository’
  • 然后 push

本地删除文件#

将删除的文件从git跟踪列表移除

  • git rm file
  • git rm -r folder
  • git rm –cached file # –cached 表示只从git跟踪列表中删除文件,而不删除本地文件
  • git push # 如果是第一次删除可能需要设置删除标记 git push –set-upstream origin 分支名

与远程仓库关联与取消#

  • git remote add origin git@github.com:git_username/repository_name.git # 关联
  • git remote remove origin # 取消

make#

makefile 核心是对依赖的管理,通过列出目标文件的依赖,整理出一个依赖关系.当执行make的是时候,会从目标文件开始向上,依次检查依赖关系,如果依赖更新,则重新执行一遍对应的流程 

hello:
  echo "Hello World!"

这个只有目标,没有依赖,所以每次执行make的时候,都会重新构建 hello 这个目标文件 

hello: 
  echo "Hello World!" < hello

第二次执行make的时候,会发现目标文件已经存在,且没有依赖,因此不会重新构建
